## 2. 枚举定义

### 2.1 优惠券类型 (CouponType)

| 值 | 名称 | 说明 |
|----|------|------|
| 1 | 满减券 | 满足最低金额后减免固定金额 |
| 2 | 折扣券 | 按比例打折，如80表示8折 |
| 3 | 无门槛券 | 无需满足条件直接减免 |

### 2.2 优惠券状态 (CouponStatus)

| 值 | 名称 | 说明 |
|----|------|------|
| 0 | 未开始 | 未到开始时间 |
| 1 | 进行中 | 可正常领取和使用 |
| 2 | 已结束 | 已过结束时间 |
| 3 | 已停用 | 管理员手动停用 |

### 2.3 用户优惠券状态 (UserCouponStatus)

| 值 | 名称 | 说明 |
|----|------|------|
| 1 | 未使用 | 已领取但未使用 |
| 2 | 已使用 | 已使用并关联订单 |
| 3 | 已过期 | 已过有效期 |

### 2.4 适用类型 (ApplyType)

| 值 | 名称 | 说明 |
|----|------|------|
| 1 | 全部商品 | 全场通用 |
| 2 | 指定分类 | 仅限指定分类商品 |
| 3 | 指定商品 | 仅限指定商品 |

### 3.2 领取优惠券

**接口说明**: 用户领取优惠券

- **请求方式**: `POST`
- **接口路径**: `/member/coupon/receive`
- **权限要求**: 需要登录
- **Content-Type**: `application/json`

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| couponId | number | 是 | 优惠券ID |

#### 请求示例

```json
{
  "couponId": 1
}
```

#### 响应数据

**成功响应**: `Coupon` 对象

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | number | 优惠券ID |
| name | string | 优惠券名称 |
| code | string | 优惠券编码 |
| type | number | 优惠券类型 |
| discountAmount | number | 优惠金额（分） |
| discountPercent | number | 折扣比例 |
| minAmount | number | 使用门槛（分） |
| status | number | 状态 |
| startTime | number | 开始时间 |
| endTime | number | 结束时间 |
| description | string | 描述 |
| applyType | number | 适用类型 |
| createTime | number | 创建时间 |

**失败响应**:

```json
{
  "code": 400,
  "message": "优惠券已领完"
}
```

#### 业务规则

1. 用户领取数量不能超过 `limitPerUser`
2. 优惠券状态必须为"进行中"
3. 剩余数量必须大于0
4. 同一用户不能重复领取同一张优惠券（除非已使用或过期）

## 5. 核心业务逻辑

### 5.1 计算优惠金额

```
if (优惠券类型 == 满减券或无门槛券) {
    优惠金额 = discountAmount
} else if (优惠券类型 == 折扣券) {
    优惠金额 = 订单金额 * (100 - discountPercent) / 100
    // 注意：折扣券可能需要设置最高抵扣金额
}

// 确保优惠金额不超过订单金额
if (优惠金额 > 订单金额) {
    优惠金额 = 订单金额
}
```

### 5.2 检查优惠券是否可用

```
1. 检查优惠券是否存在且状态为"进行中"
2. 检查当前时间在 start_time 和 end_time 之间
3. 检查用户领取数量是否超过 limitPerUser
4. 检查剩余数量是否充足（totalCount == -1 或 usedCount < totalCount）
5. 检查用户是否已领取过且未使用/未过期（根据业务规则）
```

### 5.3 检查优惠券是否满足订单条件

```
1. 检查优惠券状态为"未使用"
2. 检查当前时间在有效期内
3. 检查订单金额 >= minAmount（如果minAmount > 0）
4. 检查商品是否在适用范围内（根据applyType和applyCategoryIds/applyGoodsIds）
```
