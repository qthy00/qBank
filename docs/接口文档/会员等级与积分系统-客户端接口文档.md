# 会员等级与积分系统 - 客户端接口文档

## 概述

本文档描述了青田好游科技开发平台会员模块的客户端接口（App端），包括会员等级体系和积分系统的相关接口。

---

## 接口清单

### 一、会员等级体系接口

#### 1. 获取当前会员等级信息

**请求**
```
GET /app-api/member/user/get
```

**说明**
用户基本信息接口已包含当前会员等级信息，无需单独接口。

#### 2. 获取积分明细

**请求**
```
GET /app-api/member/point/record/page?pageNo=1&pageSize=10&addStatus=true
```

**说明**
获取当前登录用户的积分变动记录，支持筛选收入/支出。

**请求参数**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| pageNo | Integer | 否 | 页码，默认1 |
| pageSize | Integer | 否 | 每页数量，默认10 |
| addStatus | Boolean | 否 | 筛选条件：true-收入，false-支出，null-全部 |
| createTime | DateTime[] | 否 | 时间范围筛选 [开始时间, 结束时间] |

**响应示例**
```json
{
  "code": 0,
  "data": {
    "list": [
      {
        "id": 1001,
        "title": "每日签到",
        "description": "连续签到7天奖励",
        "point": 20,
        "createTime": "2026-03-26 09:00:00"
      },
      {
        "id": 1002,
        "title": "积分兑换",
        "description": "兑换优惠券",
        "point": -100,
        "createTime": "2026-03-25 14:30:00"
      }
    ],
    "total": 50
  }
}
```

**字段说明**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 记录ID |
| title | String | 积分变动标题 |
| description | String | 积分变动描述 |
| point | Integer | 变动积分（正数表示收入，负数表示支出） |
| createTime | DateTime | 记录创建时间 |

---

#### 3. 积分获取规则说明

**说明**
积分获取规则为系统配置，目前通过以下方式获取积分：

| 获取方式 | 积分值 | 说明 |
|----------|--------|------|
| 每日签到 | 5-20 | 基础5分，连续签到递增 |
| 完善资料 | 50 | 首次完善个人信息 |
| 绑定手机 | 30 | 首次绑定手机号 |
| 消费返积分 | 订单金额×1% | 每消费1元返1积分 |
| 邀请好友 | 100 | 好友注册并完成首单 |

> **注意**：具体积分规则以后端配置为准，客户端可通过管理后台配置的规则展示。

## 示例代码

### 1. 获取用户等级信息

```javascript
// 获取用户基本信息（包含等级）
async function getUserInfo() {
  const res = await axios.get('/app-api/member/user/get')
  const userInfo = res.data.data

  console.log('当前等级:', userInfo.level?.name)
  console.log('经验值:', userInfo.experience)
  console.log('积分:', userInfo.point)

  return userInfo
}
```

### 2. 获取等级列表

```javascript
// 获取所有等级
async function getLevelList() {
  const res = await axios.get('/app-api/member/level/list')
  const levels = res.data.data

  // 按等级排序
  levels.sort((a, b) => a.level - b.level)

  // 计算每级所需经验
  levels.forEach((item, index) => {
    if (index > 0) {
      item.needExperience = item.experience - levels[index - 1].experience
    } else {
      item.needExperience = item.experience
    }
  })

  return levels
}
```

### 3. 获取经验记录

```javascript
// 获取经验记录
async function getExperienceRecord(pageNo = 1, pageSize = 10) {
  const res = await axios.get('/app-api/member/experience-record/page', {
    params: { pageNo, pageSize }
  })
  return res.data.data
```

### 4. 获取积分明细

```javascript
// 获取积分记录（带筛选）
async function getPointRecord(options = {}) {
  const { pageNo = 1, pageSize = 10, addStatus = null, startTime, endTime } = options

  const params = { pageNo, pageSize }
  if (addStatus !== null) params.addStatus = addStatus
  if (startTime && endTime) {
    params.createTime = [startTime, endTime]
  }

  const res = await axios.get('/app-api/member/point/record/page', { params })
  return res.data.data
}

// 获取收入记录
async function getIncomeRecord(pageNo = 1) {
  return getPointRecord({ pageNo, addStatus: true })
}

// 获取支出记录
async function getExpenseRecord(pageNo = 1) {
  return getPointRecord({ pageNo, addStatus: false })
}
```

### 5. 等级进度计算

```javascript
// 计算等级进度
function calculateLevelProgress(userInfo, levels) {
  const currentLevel = userInfo.level
  const currentExp = userInfo.experience

  // 找到当前等级在列表中的位置
  const currentIndex = levels.findIndex(l => l.level === currentLevel?.level)

  if (currentIndex === -1 || currentIndex === levels.length - 1) {
    // 已满级
    return {
      currentLevel,
      nextLevel: null,
      progress: 100,
      currentExp,
      needExp: 0,
      isMaxLevel: true
    }
  }

  const nextLevel = levels[currentIndex + 1]
  const currentLevelExp = levels[currentIndex].experience
  const nextLevelExp = nextLevel.experience

  const needExp = nextLevelExp - currentExp
  const levelGap = nextLevelExp - currentLevelExp
  const progress = ((currentExp - currentLevelExp) / levelGap) * 100

  return {
    currentLevel,
    nextLevel,
    progress: Math.min(progress, 100),
    currentExp,
    needExp,
    isMaxLevel: false
  }
}
```

## 注意事项

1. **登录校验**：所有接口需要用户登录后才能访问
2. **缓存策略**：等级列表可缓存，积分和经验记录建议实时获取
3. **分页加载**：积分和经验记录支持分页，建议每页10-20条
4. **积分变动**：积分收入为正数，支出为负数
5. **等级升级**：等级升级由系统自动计算，当经验值达到要求时自动升级
