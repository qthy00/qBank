## 需求描述

根据后端接口文档 `docs/接口文档/会员等级与积分系统-客户端接口文档.md`，开发会员等级体系和积分系统前端功能。

### 业务背景
- 会员等级体系用于激励用户持续学习，根据经验值自动升级
- 积分系统用于奖励用户活跃行为，可兑换优惠券等权益
- 等级和积分信息已集成在用户基本信息接口中

### 约束条件
- 技术栈：Nuxt 4 + Vue 3 + TypeScript + Element Plus + Pinia

- 需要使用member布局（用户中心侧边栏）
- 支持分页和筛选功能

## 方案设计

### 最终选择

**选定方案**：按模块化开发

**开发顺序**：
1. 先开发会员等级体系（方案A）
2. 再开发积分系统（方案B）
3. 两个模块共用用户Store中的基础信息

**选择理由**：
1. 两个功能相对独立，可以并行设计
2. 会员等级数据量较小，优先完成
3. 积分系统涉及分页筛选，复杂度稍高

---

## 技术调研

### 接口清单

| 接口 | 方法 | 路径 | 用途 |
|------|------|------|------|
| 获取用户信息 | GET | `/member/user/get` | 包含等级和积分 |
| 获取等级列表 | GET | `/member/level/list` | 展示所有等级权益 |
| 获取经验记录 | GET | `/member/experience-record/page` | 经验变动历史 |
| 获取积分明细 | GET | `/member/point/record/page` | 积分收支记录 |

### 数据模型

```typescript
/* 会员等级 */
interface MemberLevelVO {
  name: string
  level: number
  experience: number
  discountPercent: number
  icon: string
  backgroundUrl: string
}

/* 用户等级信息（嵌套在UserVO中） */
interface UserLevelVO {
  id: number
  name: string
  level: number
  icon: string
}

/* 经验记录 */
interface ExperienceRecordVO {
  title: string
  experience: number
  description: string
  createTime: string
}

/* 积分记录 */
interface PointRecordVO {
  id: number
  title: string
  description: string
  point: number
  createTime: string
}
```
