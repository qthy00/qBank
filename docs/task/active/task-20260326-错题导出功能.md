## 需求描述

实现错题导出功能，用户可从错题本中选择错题导出为PDF或Word格式，支持按条件筛选导出。

### 业务背景
- 用户需要线下复习错题，导出功能可以方便用户打印或在其他设备查看
- 导出的错题应包含题目、正确答案、用户答案、解析等完整信息
- 支持按时间、题型、掌握状态等条件筛选

### 功能目标
- [x] 错题导出筛选界面（时间范围、题型、掌握状态）
- [x] 支持PDF格式导出
- [x] 支持Word格式导出
- [x] 导出历史记录（支持）
- [ ] 导出预览功能（待后续优化）

### 约束条件
- 技术栈：Nuxt 4 + Vue 3 + TypeScript + Element Plus
- 导出文件由后端生成，前端提供筛选条件和下载
- 遵循项目现有代码规范

## 方案设计

### 方案讨论

#### 方案A：纯前端生成
- **描述**：使用jsPDF/html2doc等库在浏览器生成文件
- **优点**：
  - 无需后端配合，响应快
  - 前端独立可控
- **缺点**：
  - 复杂排版困难
  - PDF中文支持差
  - 大文件性能问题
- **适用场景**：简单导出、快速原型

#### 方案B：后端生成（推荐）
- **描述**：后端生成文件，前端提供筛选条件，返回下载链接
- **优点**：
  - 排版精确，支持复杂格式
  - 中文支持好
  - 性能可控
- **缺点**：
  - 需要后端开发
  - 异步处理需等待
- **适用场景**：正式生产环境

### 最终选择

**选定方案**：方案B - 后端生成

**选择理由**：
1. 排版效果更有保障
2. 与项目现有架构一致（其他导出功能也采用后端生成）
3. 减轻前端复杂度

## 技术调研

### 技术选型

| 技术/框架 | 版本 | 优势 | 劣势 | 是否采用 | 备注 |
| --- | --- | --- | --- | --- | --- |
| jsPDF | 最新 | 纯前端生成 | 中文支持差，排版困难 | ❌ | 不适用 |
| html2doc | 最新 | 简单HTML转Word | 复杂样式不支持 | ❌ | 不适用 |
| 后端生成 | - | 排版精确 | 需要后端配合 | ✅ | 选择此方案 |
| Element Plus Dialog | - | 模态框展示 | - | ✅ | 筛选弹窗 |

### 调研结论

**推荐方案**：后端生成PDF/Word文件

**关键依赖**：
- 后端需要提供 `/member/mistakes/export` 接口
- 支持参数：exportType(pdf/word), filters(筛选条件)
- 返回：文件下载URL或文件流

**风险评估**：
- 后端接口开发需要时间
- 可先开发前端界面，使用Mock数据测试
