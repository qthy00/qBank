| 序号 | 接口名称 | 请求方法 | 接口路径 | 说明 |

## 2. 接口详情

### 2.1 全局搜索

**接口信息**
- **请求方法**: `GET`
- **接口路径**: `/app-api/cms/search`
- **功能说明**: 根据关键词搜索内容，支持按类型筛选，返回分页结果

**请求参数**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| keyword | string | 是 | 搜索关键词 |
| type | string | 否 | 搜索类型: all-全部(默认), qbank-题库, question-题目, article-文章 |
| categoryId | number | 否 | 分类ID筛选 |
| pageNo | number | 否 | 页码，默认1 |
| pageSize | number | 否 | 每页数量，默认10 |

**请求示例**
```
GET /app-api/cms/search?keyword=数学&type=all&pageNo=1&pageSize=10
```

**响应数据结构**

```typescript
{
  code: number,           // 状态码
  msg: string,            // 消息
  data: {
    list: SearchResultItem[],    // 搜索结果列表
    groups: SearchResultGroup[], // 按类型分组结果
    total: number,               // 总数量
    pageNo: number,              // 当前页
    pageSize: number             // 每页数量
  }
}
```

**响应字段说明 - SearchResultItem**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | number | 结果ID |
| type | string | 结果类型: qbank/question/article |
| title | string | 标题 |
| summary | string | 摘要/描述 |
| categoryName | string | 分类名称 |
| createTime | string | 创建时间 |

**题库结果扩展字段 (type='qbank')**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| name | string | 题库名称 |
| questionCount | number | 题目数量 |
| price | number | 价格 |
| coverUrl | string | 封面图URL |

**题目结果扩展字段 (type='question')**

| questionType | number | 题目类型代码 |

### 2.2 搜索建议

**接口信息**
- **请求方法**: `GET`
- **接口路径**: `/app-api/cms/search/suggest`
- **功能说明**: 根据输入的关键词返回搜索建议列表，用于搜索框联想提示

**请求参数**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| keyword | string | 是 | 输入的关键词（至少2个字符） |
| type | string | 否 | 指定搜索类型: qbank/question/article，默认为all |

**请求示例**
```
GET /app-api/cms/search/suggest?keyword=数学&type=all
```

**响应数据结构**

```typescript
{
  code: number,
  msg: string,
  data: {
    list: SearchSuggestItem[]   // 建议列表
  }
}
```

**SearchSuggestItem 字段说明**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| text | string | 建议文本（完整搜索词） |
| type | string | 建议类型: qbank/question/article |
| highlightText | string | 高亮显示文本（关键词部分加标记） |
| searchCount | number | 该词的搜索次数（用于排序） |

**响应示例**
```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "list": [
      {
        "text": "数学题库",
        "type": "qbank",
        "highlightText": "<em>数学</em>题库",
        "searchCount": 1250
      },
      {
        "text": "数学题目",
        "type": "question",
        "highlightText": "<em>数学</em>题目",
        "searchCount": 890
      },
      {
        "text": "高中数学",
        "type": "qbank",
        "highlightText": "高中<em>数学</em>",
        "searchCount": 756
      }
    ]
  }
}
```

### 2.4 高级搜索

**接口信息**
- **请求方法**: `POST`
- **接口路径**: `/app-api/cms/search/advanced`
- **功能说明**: 支持复杂筛选条件的搜索，如价格区间、难度、排序等

**请求参数 (Body)**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| keyword | string | 是 | 搜索关键词 |
| type | string | 否 | 搜索类型，默认all |
| categoryId | number | 否 | 分类ID |
| pageNo | number | 否 | 页码，默认1 |
| pageSize | number | 否 | 每页数量，默认10 |
| filter | object | 否 | 筛选条件对象 |
| filter.categoryId | number | 否 | 分类ID（与外层categoryId二选一） |
| filter.difficulty | number | 否 | 难度等级(1-5) |
| filter.priceRange | array | 否 | 价格区间 [min, max] |
| filter.sort | string | 否 | 排序方式: relevance-相关度/newest-最新/popular-热门/price_asc-价格升序/price_desc-价格降序 |

**请求示例**
```json
{
  "keyword": "数学",
  "type": "qbank",
  "pageNo": 1,
  "pageSize": 10,
  "filter": {
    "categoryId": 5,
    "difficulty": 3,
    "priceRange": [0, 50],
    "sort": "popular"
  }
}
```

**响应数据结构**
与【2.1 全局搜索】相同

## 3. 技术要求

### 3.1 搜索功能要求

1. **关键词匹配字段**
   - 题库: name(名称)、summary(简介)
   - 题目: title(标题)、content(内容)
   - 文章: title(标题)、content(内容)、summary(摘要)

2. **搜索结果排序优先级**
   - 默认: 相关度 > 热度 > 创建时间
   - 支持按参数指定排序方式

3. **搜索结果高亮**
   - 建议对匹配的关键词添加高亮标记 `<em>关键词</em>`

4. **分页规范**
   - 默认每页10条，最大50条
   - 返回 total 总数量，便于前端计算页数

### 3.2 性能要求

| 指标 | 要求 |
|------|------|
| 搜索响应时间 | < 500ms |
| 搜索建议响应时间 | < 100ms |
| 热门搜索缓存 | 5分钟 |
| 搜索结果缓存 | 可配置 |

### 3.3 错误码

| 错误码 | 说明 |
|--------|------|
| 0 | 成功 |
| 400 | 请求参数错误 |
| 500 | 服务器内部错误 |

## 4. 数据库设计建议

### 4.1 搜索记录表 (search_record)

用于统计热门搜索、记录用户搜索行为

```sql
CREATE TABLE search_record (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    keyword VARCHAR(100) NOT NULL COMMENT '搜索关键词',
    type VARCHAR(20) COMMENT '搜索类型',
    user_id BIGINT COMMENT '用户ID（可选）',
    result_count INT DEFAULT 0 COMMENT '返回结果数量',
    ip VARCHAR(50) COMMENT '用户IP',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    INDEX idx_keyword (keyword),
    INDEX idx_create_time (create_time)
) COMMENT='搜索记录表';
```

### 4.2 热门搜索表 (hot_search)

预计算热门搜索词，定时更新

```sql
CREATE TABLE hot_search (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    keyword VARCHAR(100) NOT NULL COMMENT '关键词',
    search_count INT DEFAULT 0 COMMENT '搜索次数',
    sort INT DEFAULT 0 COMMENT '排序',
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    UNIQUE KEY uk_keyword (keyword)
) COMMENT='热门搜索表';
```
