## 需求描述

实现题目搜索功能，支持在题库模块内按关键词、题型、难度搜索题目，支持高级筛选。

### 业务背景
- 用户需要快速定位题库中的特定题目
- 支持按多种维度筛选题目（题型、难度、知识点）
- 与全局搜索互补，提供更专业的题目搜索体验

### 功能目标
- [ ] 题目搜索页面（搜索框 + 筛选条件 + 结果列表）
- [ ] 按关键词搜索题目
- [ ] 按题型筛选（单选、多选、判断等）
- [ ] 按难度筛选（简单、中等、困难）
- [ ] 按知识点筛选
- [ ] 搜索结果分页展示
- [ ] 支持收藏/取消收藏题目
- [ ] 点击题目进入详情

### 约束条件
- 使用Element Plus组件
- 遵循项目设计规范
- 复用现有题目卡片组件
- 支持响应式布局

---

## 方案设计

### 方案讨论

#### 方案A：独立题目搜索页面
- **描述**：创建独立的 `/question/search` 页面，专门用于题目搜索
- **优点**：
  - 页面功能专一，用户体验好
  - 筛选条件可以更丰富
  - 与题库详情页分离，避免干扰
- **缺点**：
  - 需要新增路由
  - 与全局搜索结果页有一定重复
- **适用场景**：用户需要专注搜索题目的场景
- **技术栈**：Nuxt页面 + Element Plus + 现有API

#### 方案B：在题库详情页增加题目搜索
- **描述**：在题库详情页的章节列表上方增加搜索框
- **优点**：
  - 上下文清晰，用户在特定题库内搜索
  - 与题库内容关联性强
- **缺点**：
  - 只能搜索当前题库内的题目
  - 筛选条件受限
- **适用场景**：用户浏览特定题库时搜索题目
- **技术栈**：复用现有题库详情页

### 最终选择

**选定方案**：方案A

**选择理由**：
1. 题目搜索是独立功能，不应局限于某个题库
2. 可以提供更完整的筛选条件
3. 搜索体验更专注
4. 与全局搜索形成互补

**权衡考量**
- 优先搜索体验，牺牲一点开发成本
- 后续可在题库详情页复用搜索组件

**备选方案**
- 如果后续需要在特定题库内搜索，可以在题库详情页引入此搜索组件
