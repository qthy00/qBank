## 需求描述

实现题目收藏功能，允许用户在练习和考试过程中收藏题目，并在个人中心的收藏夹中管理收藏的题目。

### 业务背景
- 用户在练习过程中遇到重点/难点题目需要标记
- 需要针对性地复习收藏的题目
- 与现有的题库收藏形成完整功能闭环

### 功能目标
- [ ] 题目收藏/取消收藏按钮（练习页面、考试页面、题目详情）
- [ ] 收藏状态实时显示
- [ ] 我的收藏页面支持题目分类管理
- [ ] 收藏夹支持按题库筛选

### 约束条件
- 复用现有收藏API架构（contentType区分类型）
- 使用Element Plus组件
- 遵循项目设计规范
- 支持响应式布局

---

## 方案设计

### 方案讨论

#### 方案A：扩展现有收藏系统
- **描述**：复用现有 `contentType` 字段，新增 `question` 类型
- **优点**：
  - 复用现有API接口，开发成本低
  - 数据结构统一，维护简单
  - 与现有收藏功能无缝集成
- **缺点**：
  - 题目收藏可能需要额外的题目信息展示
- **适用场景**：题目收藏与工具收藏逻辑相似
- **技术栈**：复用现有 `favorites` API

#### 方案B：独立题目收藏模块
- **描述**：单独创建题目收藏API和表结构
- **优点**：
  - 可以针对题目特性定制功能
  - 更好的扩展性
- **缺点**：
  - 重复开发收藏基础功能
  - 增加系统复杂度
- **适用场景**：题目收藏有特殊业务逻辑

### 最终选择

**选定方案**：方案A

**选择理由**：
1. 现有收藏系统已支持通过 `contentType` 区分类型
2. 收藏/取消/列表查询逻辑完全一致
3. 减少重复代码，降低维护成本
4. 开发周期短（半天内可完成）

**技术实现**
- `contentType='question'` 表示题目收藏
- 复用现有 `/cms/member/favorites` 接口
- 扩展「我的收藏」页面支持题目类型展示
