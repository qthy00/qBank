# /init-docs - 初始化业务文档

作为项目文档初始化助手，根据用户选项的模式初始化项目文档。

支持三种使用方式：
- `/init-docs` - 交互式选择模式
- `/init-docs 实现用户反馈管理功能` - 基于描述智能初始化
- `/init-docs docs/brainstorm-用户反馈.md` - 基于需求文档智能初始化

---

## 第零步：参数检测（在模式选择前执行）

检查用户是否在 `/init-docs` 后附带了参数。

### 检测规则

| 参数类型     | 判断条件                             | 处理方式                |
|:---------|:---------------------------------|:--------------------|
| **无参数**  | `/init-docs` 后无内容                | 进入第一步（交互式选择模式）      |
| **文件路径** | 参数以 `.md` 结尾，或包含 `/` 或 `\` 路径分隔符 | 进入**模式三：基于需求文档初始化** |
| **文本描述** | 其他情况（中文描述、功能名称等）                 | 进入**模式三：基于描述初始化**   |

### 文件路径检测

如果参数看起来像文件路径：
1. 检查文件是否存在
2. 如果存在，读取文件内容，进入模式三
3. 如果不存在，提示文件未找到，建议：
   ```
   未找到文件：｛路径｝

   你可以：
   1. 先运行头脑风暴生成需求文档：直接描述你想实现的功能
   2. 直接使用描述初始化：/init-docs ｛功能描述｝
   3. 交互式选择模式：/init-docs
   ```

---

## 第一步：询问用户选择模式

> **注意**：仅在无参数时执行此步骤。有参数时直接跳到模式三。

### 上下文智能检测

在展示选项前，先检查**当前对话上下文**

- 如果当前对话中**已经进行过头脑风暴**（对话中包含"方案设计"、"方案对比"、"推荐方案"、"实施步骤"）
  ```
  检测到当前对话中有头脑风暴产出的方案设计。

  是否基于该方案直接初始化项目文档？（推荐）
  - 输入"是"或"确认" -> 进入模式三，自动提取头脑风暴结论
  - 输入"否" -> 显示完整模式选择菜单
  ```

### 模式选择菜单

```
请选择初始化模式：

1. 从零开始创建
   - 扫描现有业务模块
   - 创建空白项目文档
   - 适用于：已有代码，需要补全文档

2. 基于头脑风暴结果创建
   - 自动提取头脑风暴的方案设计
   - 生成完整的需求文档和待办清单
   - 适用于：刚完成方案讨论

3. 基于功能描述创建
   - 智能分析功能需求
   - 自动生成技术方案和任务清单
   - 适用于：明确知道要实现什么功能

4. 基于需求文档创建
   - 读取已有的需求文档
   - 提取需求生成项目文档
   - 适用于：已有详细需求文档

请输入选项 (1/2/3/4):
```

## 模式一：从零开始创建

### 第一步：扫描业务模块

根据项目类型扫描业务代码：

#### SpringBoot项目
```bash
# 查找所有Controller
find . -path "*/src/main/java/*" -name "*Controller.java" | grep -v test

# 提取模块名称（从包路径）
# 如：com.example.module.user -> user模块
```

#### Vue项目
```bash
# 查找views目录下的页面
find ./src/views -name "*.vue" -type f 2>/dev/null | head -20

# 查找components目录
find ./src/components -name "*.vue" -type f 2>/dev/null | head -20
```

#### Nuxt项目
```bash
# 查找pages目录下的页面
find ./app/pages -name "*.vue" -type f 2>/dev/null | head -20

# 查找composables
find ./app/composables -name "*.ts" -type f 2>/dev/null | head -20
```

#### UniApp项目
```bash
# 查找pages目录
find ./src/pages -name "*.vue" -type f 2>/dev/null | head -20
```

### 第二步：生成文档内容

**项目状态.md**：
- 项目名称：从package.json/pom.xml提取
- 项目类型：识别结果
- 整体进度：0%
- 模块概览：扫描到的模块列表

**待办清单.md**：
- 扫描代码中的TODO/FIXME作为初始待办
- 按文件路径分类

**需求文档.md**：
- 项目基本信息
- 技术栈信息（从依赖文件提取）
- 功能模块列表

### 第三步：创建文档文件

```bash
mkdir -p docs
touch docs/项目状态.md
touch docs/待办清单.md
touch docs/需求文档.md
```

### 第四步：输出结果

```
✅ 项目文档初始化完成！

已创建以下文档：
- docs/项目状态.md (扫描到 ｛N｝ 个模块)
- docs/待办清单.md (提取了 ｛N｝ 个TODO)
- docs/需求文档.md (识别了技术栈：｛技术栈｝)

扫描到的业务模块：
- 模块1: ｛功能数｝ 个功能
- 模块2: ｛功能数｝ 个功能
...

下一步建议：
- 运行 /update-status 更新项目状态
- 运行 /next 获取下一步开发建议
```

## 模式二：基于头脑风暴结果创建

### 第一步：提取头脑风暴结论

从对话历史中提取：
- 项目名称/描述
- 功能需求列表
- 技术选型方案
- 实施步骤
- 风险评估

### 第二步：生成需求文档

根据提取的信息填充：
- 项目描述
- 技术栈选型
- 功能需求（REQ-001, REQ-002...）
- 非功能需求
- 接口定义
- 项目目录结构

### 第三步：生成待办清单

根据实施步骤生成：
- 高优先级任务（核心功能）
- 中优先级任务（次要功能）
- 低优先级任务（优化项）

### 第四步：生成项目状态

- 项目阶段：开发中
- 整体进度：0%
- 模块概览：根据功能模块划分
- 里程碑：根据实施步骤设定

### 第五步：输出结果

```
✅ 基于头脑风暴结果初始化完成！

已提取并生成：
- 需求文档.md (｛N｝ 个功能需求，｛N｝ 个技术选型)
- 待办清单.md (｛N｝ 个任务，含优先级)
- 项目状态.md (｛N｝ 个模块，｛N｝ 个里程碑)

提取的核心需求：
- REQ-001: ｛功能描述｝
- REQ-002: ｛功能描述｝
...

推荐技术方案：｛技术方案｝
预计完成时间：｛估算时间｝

下一步：运行 /next 获取具体开发建议
```

## 模式三：基于功能描述创建

### 第一步：智能分析功能描述

分析用户提供的功能描述：
- 提取核心功能点
- 识别业务实体（如：用户、订单、商品）
- 推测需要的CRUD操作
- 识别可能的关联功能

### 第二步：生成技术方案

根据项目类型推荐技术方案：

#### 如果是 SpringBoot 后端
- 推荐数据库表设计
- 推荐实体类结构
- 推荐接口设计

#### 如果是 Vue 前端
- 推荐页面结构
- 推荐组件划分
- 推荐路由设计

#### 如果是 Nuxt 跨端
- 推荐渲染模式（SSR/SSG/SPA）
- 推荐目录结构
- 推荐组件设计

#### 如果是 UniApp 跨端
- 推荐页面配置
- 推荐多端适配方案
- 推荐原生能力使用

### 第三步：生成完整文档

生成需求文档、待办清单、项目状态三个文档。

### 第四步：输出结果

```
✅ 基于功能描述初始化完成！

功能分析：
- 核心功能: ｛描述｝
- 业务实体: ｛实体1｝、｛实体2｝
- 预估工作量: ｛估算｝

已生成文档：
- 需求文档.md
- 待办清单.md
- 项目状态.md

下一步建议：
- 如需调整需求，直接编辑 docs/需求文档.md
- 运行 /next 开始第一个任务
```

## 模式四：基于需求文档创建

### 第一步：读取需求文档

读取用户指定的需求文档文件。

### 第二步：提取需求信息

- 项目基本信息
- 功能需求列表
- 技术栈选型
- 接口定义

### 第三步：生成待办清单

根据功能需求生成任务列表：
- 每个REQ对应一个或多个任务
- 按优先级分类
- 设置合理的截止日期

### 第四步：生成项目状态

- 项目阶段：开发中
- 整体进度：0%
- 模块概览：根据需求模块划分

### 第五步：输出结果

```
✅ 基于需求文档初始化完成！

来源文档: ｛文件路径｝

提取信息：
- 项目名称: ｛名称｝
- 功能需求: ｛N｝ 个
- 技术栈: ｛技术栈｝

已生成文档：
- 需求文档.md (复制并格式化)
- 待办清单.md (｛N｝ 个任务)
- 项目状态.md (｛N｝ 个模块)

下一步：运行 /next 获取开发建议
```

## 通用：技术栈识别逻辑

根据项目文件自动识别技术栈：

### SpringBoot
```xml
<!-- pom.xml -->
<parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>{version}</version>
</parent>
```

### Vue
```json
// package.json
{
  "dependencies": {
    "vue": "^3.x",
    "vue-router": "^4.x"
  }
}
```

### Nuxt
```json
// package.json
{
  "dependencies": {
    "nuxt": "^3.x"
  }
}
```

### UniApp
```json
// package.json
{
  "dependencies": {
    "@dcloudio/uni-app": "^3.x"
  }
}
```

## 通用：TODO扫描逻辑

```bash
# 扫描代码中的TODO/FIXME
grep -r "TODO\|FIXME\|XXX" --include="*.java" --include="*.ts" --include="*.vue" --include="*.tsx" .
```

提取格式：
- 文件路径
- 行号
- TODO内容
- 分类（技术债务/功能待实现/待优化）
