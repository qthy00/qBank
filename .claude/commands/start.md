# /start - 新窗口快速了解项目

作为项目引导助手，请帮我快速了解这个项目的当前状态。

---

## 重要：读取框架配置

首先读取 `.claude/framework-config.json`，了解哪些是框架模块（不统计）：哪些是业务模块（需统计）。

如果没有该文件，使用默认配置：
```json
{
  "frameworkModules": [
    "ruoyi-system",
    "ruoyi-common",
    "ruoyi-framework"
  ],
  "businessModules": []
}
```

---

## 第一步：项目基本信息识别

### 1.1 识别项目类型

通过检测项目特征文件判断技术栈：

| 检测文件 | 项目类型 | 说明 |
|---------|---------|------|
| `pom.xml` + `src/main/java` | SpringBoot | Java后端项目 |
| `package.json` + `vite.config.ts` + 无 `nuxt.config.ts` | Vue | Vue3前端项目 |
| `nuxt.config.ts` | Nuxt | Nuxt跨端项目 |
| `package.json` + `manifest.json` + `pages.json` | UniApp | 跨端小程序/App |

**输出示例**：
```
项目类型：Nuxt 3 + Vue 3 + TypeScript
技术栈：
  - 框架：Nuxt 3.10.x
  - UI库：Element Plus 2.x
  - 状态：Pinia 2.x
  - 样式：UnoCSS
```

### 1.2 查看Git提交历史

**Git 提交过滤规则**（与 /next、/progress 一致）
```bash
git log -5 --format="%H|%an|%cn|%s" --no-merges
```

| 过滤条件               | 识别方式                                            | 分类              |
|:-------------------|:------------------------------------------------|:----------------|
| Cherry-pick 提交     | commit message 包含 `cherry picked from`          | 同步提交            |
| 上游同步提交             | commit message 以 `sync:` 或 `upstream:` 开头       | 同步提交            |
| Author ≠ Committer | author 和 committer 名字不同                         | 可能是 cherry-pick |
| 文档/配置提交            | commit message 以 `docs:` `chore:` `style:`   开头 | 非功能提交           |

**输出**：只展示过滤后的真实开发提交

## 第二步：智能检测项目状态

### 2.1 检测Git状态

```bash
git status --short
git branch --show-current
git log --oneline --graph -10
```

**输出**：
- 当前分支
- 是否有未提交修改
- 最近分支活动

### 2.2 统计业务模块

**根据 framework-config.json 排除框架模块**后统计：

```
业务模块统计：
- 总模块数：｛N｝个
- 框架模块（排除）：｛N｝个
- 业务模块：｛N｝个
- 模块列表：
  - module-a: ｛功能数｝个功能
  - module-b: ｛功能数｝个功能
```

### 2.3 检测项目文档

检查是否存在以下文档：
- `docs/项目状态.md`
- `docs/待办清单.md`
- `docs/需求文档.md`

**输出**：
```
项目文档状态：
✅ 项目状态.md (最后更新: 2024-01-15)
⚠️ 待办清单.md (缺失 - 运行 /init-docs 初始化)
✅ 需求文档.md (3个功能需求)
```

### 2.4 检测活跃任务

扫描 `docs/tasks/active/` 目录：
- 活跃任务数量
- 每个任务的状态和进度
- 最后更新时间

**输出**：
```
活跃任务 (｛N｝个):
- 任务A: 进行中 (65%) - 2天前更新
- 任务B: 暂停 (30%) - 7天未更新 ⚠️
```

### 2.5 检测待办事项

读取 `docs/待办清单.md`：
- 高优先级待办数量
- 中优先级待办数量
- 低优先级待办数量
- 本周目标

**输出**：
```
待办事项：
- 高优先级: ｛N｝个
- 中优先级: ｛N｝个
- 低优先级: ｛N｝个
- 本周目标: ｛目标描述｝
```

### 2.6 检测代码统计

```bash
# 统计代码行数（根据项目类型调整）
find . -name "*.vue" -o -name "*.ts" -o -name "*.tsx" | xargs wc -l 2>/dev/null | tail -1
```

## 第四步：提供快速操作建议

根据检测结果，推荐最可能的下一步：

1. **如果项目文档缺失** → 推荐 `/init-docs`
2. **如果有活跃任务且有阻塞** → 提示阻塞问题
3. **如果有进行中任务** → 显示任务进度
4. **如果待办清单有高优先级** → 提示高优先级待办
5. **如果刚完成头脑风暴** → 提示初始化文档

---

## 执行流程

1. 读取 `.claude/framework-config.json`
2. 识别项目类型和技术栈
3. 获取Git提交历史（过滤非开发提交）
4. 检测Git状态（分支、修改）
5. 统计业务模块（排除框架模块）
6. 检测项目文档状态
7. 扫描活跃任务
8. 读取待办清单
9. 统计代码行数
10. **输出项目概览**
11. 提供快速操作建议
