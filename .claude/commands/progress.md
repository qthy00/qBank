# /progress - 项目进度报告

作为项目进度分析助手，综合分析项目全貌后，输出**结构化的全面进度报告**。

---

## 第一步：收集项目全貌（6个数据源，并行执行）

### 1.1 Git 提交分析（过滤非开发提交）

```bash
# 获取最近 20 条提交，包含 author/committer/message 信息
git log -20 --format="%H|%an|%cn|%s" --no-merges
```

**过滤规则** （以下提交不计入开发活动统计）：

| 过滤条件               | 识别方式                                            | 分类              |
|:-------------------|:------------------------------------------------|:----------------|
| Cherry-pick 提交     | commit message 包含 `cherry picked from`          | 同步提交            |
| 上游同步提交             | commit message 以 `sync:` 或 `upstream:` 开头       | 同步提交            |
| Author ≠ Committer | author 和 committer 名字不同                         | 可能是 cherry-pick |
| 文档/配置提交            | commit message 以 `docs:` `chore:` `style:`   开头 | 非功能提交           |

**输出**：
- 开发提交数量和列表（过滤后）
- 同步提交数量（单独统计）
- 非功能提交数量（docs/chore/style）
- 最近开发方向（根据 commit scope 归类）

### 1.2 活跃任务文档

```bash
# 扫描活跃任务
ls docs/tasks/active/ 2>/dev/null || echo "目录不存在"
```

对每个活跃任务文档，**必须深度读取以下区域**：

| 读取区域        | 提取信息                          | 用途           |
|:------------|:------------------------------|:-------------|
| 文件头部（状态/时间） | 状态（进行中/暂停）、最后更新时间             | 判断是否停滞       |
| 实现步骤        | `- [ ]` 和 `- [x]` 统计，下一个未完成步骤 | 计算进度、确定下一步   |
| 当前进度        | "正在进行"、"下一步操作"、"阻塞问题"         | **直接作为推荐依据** |
| 问题记录        | 状态为"待解决" 或 "进行中"的问题           | 判断是否有阻塞      |
| 关键决策        | 技术选型、架构模式                     | 理解任务上下文      |

**关键判断**：
- 如果任务有"阻塞问题"（"待解决"） -> 推荐解决阻塞
- 如果任务 "下一步操作" 明确 -> **直接推荐任务文档中的下一步**
- 如果任务超过7天无更新 -> 提醒用户是否需要继续或暂停

### 1.3 待办清单

```bash
# 读取待办清单
cat docs/待办清单.md 2>/dev/null || echo "文件不存在"
```

提取：
- 高/中/低优先级任务数量
- 已完成任务数量
- 完成率统计
- 本周目标
- 本周已完成
- 本周计划

### 1.4 项目状态文档

```bash
# 读取项目状态
cat docs/项目状态.md 2>/dev/null || echo "文件不存在"
```

提取：
- 项目阶段
- 整体进度百分比
- 模块概览表格
- 里程碑状态
- 问题与风险
- 技术栈分布

### 1.5 需求文档

```bash
# 读取需求文档
cat docs/需求文档.md 2>/dev/null || echo "文件不存在"
```

提取：
- 功能需求实现情况（REQ-xxx 状态）
- 技术栈选型
- 已实现的验收标准比例

### 1.6 代码统计

根据项目类型执行代码统计：

```bash
# SpringBoot
find . -name "*.java" -not -path "*/test/*" | xargs wc -l 2>/dev/null | tail -1

# Vue/Nuxt/UniApp
find . -name "*.vue" -o -name "*.ts" -not -path "*/node_modules/*" | xargs wc -l 2>/dev/null | tail -1
```

统计：
- 总代码行数
- 文件数量
- 组件/页面数量
- API接口数量

## 执行流程总结

1. **并行收集数据**
   - Git提交分析
   - 活跃任务扫描
   - 待办清单读取
   - 项目状态读取
   - 需求文档读取
   - 代码统计

2. **数据分析**
   - 计算整体进度
   - 统计任务完成情况
   - 分析开发活跃度
   - 识别阻塞问题

3. **生成报告**
   - 项目概览
   - 模块进度
   - 里程碑跟踪
   - 活跃任务详情
   - 待办清单统计
   - 需求实现情况
   - 代码统计
   - Git提交统计
   - 问题与风险
   - 总结建议

4. **输出与导出**
   - 显示完整报告
   - 提供导出选项
