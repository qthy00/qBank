# 技术选型与决策指南

## 何时使用

**适用场景**
- 技术方案对比与选择
- 架构决策评估
- 实现方式选择
- 第三方库选型
- 性能优化方案选择
- 状态管理方案选择
- 部署策略选择

**不适用场景**
- 具体代码编写
- Bug调试
- 简单的事实查询

## 决策框架

### 1. 决策评估矩阵

| 评估维度 | 权重 | 说明 |
|---------|------|------|
| 功能满足度 | 30% | 是否满足业务需求 |


## 常见技术决策

### 1. 状态管理方案选择

| 方案 | 适用场景 | 优点 | 缺点 |
|-----|---------|------|------|
| **Pinia**（推荐）| 中大型应用 | Vue官方推荐、TypeScript友好、简洁 | 需要学习成本 |
| Vuex | 旧项目维护 | 成熟稳定 | Vue3已不推荐 |
| Provide/Inject | 小型应用/组件通信 | 无需引入库 | 复杂场景难以管理 |
| LocalStorage | 持久化简单数据 | 刷新不丢失 | 只能存字符串 |

**决策建议：**
- 新项目首选 **Pinia**
- 简单全局状态可用 **Provide/Inject**
- 需要持久化的用户偏好设置用 **LocalStorage + Pinia持久化插件**

### 2. 组件通信方案

| 方式 | 场景 | 代码示例 |
|-----|------|---------|
| Props/Emits | 父子组件 | 标准方式 |
| Provide/Inject | 跨层级组件 | 依赖注入 |
| Pinia | 任意组件 | 全局状态 |
| Event Bus | 任意组件（不推荐）| mitt库 |

**决策建议：**
- 优先使用 **Props/Emits** 进行父子通信
- 深层嵌套组件使用 **Provide/Inject**
- 全局状态使用 **Pinia**
- 避免使用 Event Bus（难以追踪）

### 3. 数据获取方案

| 方案 | 适用场景 | 优点 | 缺点 |
|-----|---------|------|------|
| **useHttp封装**（推荐）| 大多数场景 | 统一处理、灵活 | 需手动管理loading |
| useFetch | Nuxt SSR | 支持服务端获取 | 功能较简单 |
| useAsyncData | Nuxt 组合式 | 自动处理状态 | Nuxt专属 |
| TanStack Query | 复杂数据管理 | 缓存、重试、刷新 | 包体积增加 |

**决策建议：**
- 普通CRUD使用 **useHttp封装**
- 需要服务端渲染用 **useAsyncData**
- 复杂数据流管理考虑 **TanStack Query**

### 4. 表单处理方案

| 方案 | 适用场景 | 优点 | 缺点 |
|-----|---------|------|------|
| **Element Plus Form**（推荐）| 简单/中等表单 | 简单直接、UI统一 | 复杂表单代码多 |
| FormKit | 复杂表单 | 强大的表单功能 | 学习成本 |
| VeeValidate + Yup | 需要复杂验证 | 验证逻辑分离 | 增加复杂度 |
| 自定义hooks | 特殊需求 | 完全可控 | 开发成本高 |

**决策建议：**
- 常规表单使用 **Element Plus Form**
- 动态表单/复杂验证使用 **VeeValidate**
- 表单逻辑复杂可提取为 **Composable**

### 5. UI组件库选择

| 方案 | 适用场景 | 优点 | 缺点 |
|-----|---------|------|------|
| **Element Plus**（已选）| 后台管理系统 | 成熟、组件丰富、文档好 | 样式定制需覆盖 |
| Naive UI | 现代后台 | 美观、TypeScript友好 | 社区较小 |
| Ant Design Vue | 企业应用 | 功能强大 | 包体积大 |
| Vuetify | Material Design | 设计规范 | 风格固定 |

**决策建议：**
- 本项目已使用 **Element Plus**，保持稳定
- 如需升级或替换，考虑 **Naive UI**（Vue生态趋势）

## 性能优化方案决策

### 1. 首屏优化

| 方案 | 效果 | 实现复杂度 | 优先级 |
|-----|------|-----------|--------|
| SSR/SSG | 显著提升 | 中 | P0 |
| 代码分割 | 中等提升 | 低 | P0 |
| 图片优化 | 显著提升 | 低 | P0 |
| 预加载关键资源 | 中等提升 | 低 | P1 |

### 2. 运行时优化

| 方案 | 效果 | 适用场景 |
|-----|------|---------|
| 虚拟列表 | 大幅提升 | 大数据列表 |
| 懒加载 | 中等提升 | 图片、组件 |
| Memoization | 中等提升 | 复杂计算 |
| Debounce/Throttle | 防止卡顿 | 频繁触发事件 |

### 3. 优化决策树

```
页面加载慢？
├── 首屏慢？
│   ├── 是 → 启用SSR/SSG
│   ├── JS文件大？ → 代码分割
│   └── 图片多？ → 图片懒加载+压缩
│
├── 操作卡顿？
│   ├── 列表卡顿？ → 虚拟滚动
│   ├── 输入卡顿？ → Debounce
│   └── 渲染卡顿？ → 优化组件渲染
│
└── 内存占用高？
    ├── 组件未销毁？ → 检查生命周期
    └── 数据缓存大？ → 限制缓存大小
```

## 架构决策

### 1. 渲染策略选择

| 策略 | 优点 | 缺点 | 适用场景 |
|-----|------|------|---------|
| SSR | SEO友好、首屏快 | 服务器压力 | 内容型站点 |
| CSR | 交互体验好 | 首屏慢、SEO差 | 后台管理系统 |
| SSG | 最快、成本低 | 数据不实时 | 静态内容 |
| ISR | 平衡SSG和SSR | 配置复杂 | 内容站点 |

**本项目决策：** 混合使用
- 前台页面：SSR（SEO需要）
- 后台管理：CSR（纯交互）
- 静态页面：SSG

### 2. 模块组织方式

| 方式 | 优点 | 缺点 | 适用 |
|-----|------|------|------|
| 按类型分 | 结构清晰 | 修改需跨目录 | 小型项目 |
| 按业务分 | 内聚性高 | 类型分散 | 大型项目 |
| 混合方式 | 平衡 | 需约定 | 中型项目 |

**本项目决策：** 混合方式
```
app/
├── api/           # 按类型 - API层
├── components/    # 按类型 - 组件层
├── pages/         # 按业务 - 页面层
└── stores/        # 按类型 - 状态层
```

### 3. 错误处理策略

| 策略 | 优点 | 缺点 | 适用 |
|-----|------|------|------|
| 全局拦截 | 统一处理 | 不够灵活 | 通用错误 |
| 组件内处理 | 精准处理 | 代码重复 | 业务错误 |
| Error Boundary | UI隔离 | 无法捕获异步错误 | 渲染错误 |

**本项目决策：** 组合使用
- API错误：全局拦截 + 状态码处理
- 业务错误：组件内处理
- 渲染错误：Error Boundary（Vue errorCaptured）

## 第三方库选型

### 1. 日期处理

| 库 | 大小 | 优点 | 决策 |
|----|------|------|------|
| **dayjs** | 2KB | 小巧、Moment兼容 | ✅ 推荐 |
| date-fns | 15KB+ | 函数式、tree-shaking | 备选 |
| moment | 70KB+ | 功能全 | ❌ 废弃 |

### 2. HTTP请求

| 库 | 优点 | 决策 |
|----|------|------|
| **$fetch** | Nuxt内置、SSR友好 | ✅ 推荐 |
| axios | 功能全、拦截器 | 兼容旧项目 |
| ofetch | $fetch的基础、通用 | 跨框架 |

### 3. 工具函数

| 库 | 优点 | 决策 |
|----|------|------|
| **@vueuse/core** | Vue专属、组合式 | ✅ 推荐 |
| lodash-es | 功能全、tree-shaking | 复杂操作 |
| radash | 现代、TypeScript | 新项目 |

### 4. 图表库

| 库 | 优点 | 缺点 | 决策 |
|----|------|------|------|
| ECharts | 功能全、中文文档 | 体积大 | 复杂图表 |
| Chart.js | 轻量、简单 | 功能较少 | 简单图表 |
| D3.js | 极度灵活 | 学习曲线陡 | 定制化 |
