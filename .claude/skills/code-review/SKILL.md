执行全面的代码审查，按照以下维度检查代码质量：

## 审查维度

1. **TypeScript 规范**
   - 类型定义完整性（避免 `any`）
   - 接口命名规范（大驼峰）
   - 类型文件位置正确性

2. **Vue 3 组合式 API 规范**
   - `<script setup lang="ts">` 语法
   - Composition API 使用
   - 生命周期钩子正确使用

3. **Nuxt.js 规范**
   - SSR/CSR 模式正确性
   - 目录结构规范
   - API 调用规范

4. **HTTP/API 规范**
   - `useHttp` 使用
   - 错误处理完整性

5. **样式规范**
   - UnoCSS 原子类优先
   - CSS 变量使用
   - 注释格式正确性

6. **状态管理规范**
   - Pinia Store 命名
   - 持久化配置

7. **组件规范**
   - Props/Emits 定义
   - 组件文件组织

8. **性能优化**
   - 懒加载使用
   - 响应式数据优化

9. **安全性**
   - XSS 防护
   - Token 处理

10. **代码质量**
    - 单一职责
    - 错误处理
    - 注释清晰度

## 输出格式

按严重等级输出问题：
- 🔴 严重：影响功能、安全漏洞
- 🟡 警告：代码风格、潜在问题
- 🟢 建议：优化建议、最佳实践

包含：问题描述、文件位置、修复建议、代码示例
