# 文档中文化技能

将英文文档翻译成高质量的中文文档，保持原有格式和结构。

## 使用场景

- 将单个 Markdown 文件翻译成中文
- 批量翻译整个目录下的所有文档
- 翻译指定格式（如 .md, .txt）的文件
- 需要断点续传的大型翻译任务

## 工作流程

### 1. 分析任务

首先确认用户的翻译需求：
- **源文件**：单个文件路径 或 目录路径
- **目标位置**：输出目录（默认在源目录下创建 `zh-CN/` 子目录）
- **文件格式**：默认 `.md`，可指定其他格式
- **任务状态**：检查是否存在未完成的翻译任务

### 2. 扫描待翻译文件

使用 `Glob` 工具扫描所有匹配的文件：

```javascript
// 示例：扫描所有 .md 文件
Glob({ pattern: "**/*.md", path: sourceDir })
```

### 3. 检查翻译状态

读取进度跟踪文件 `.translation-progress.json`（如果存在）：

```json
{
  "startedAt": "2024-01-15T10:30:00Z",
  "sourceDir": "/path/to/source",
  "targetDir": "/path/to/source/zh-CN",
  "totalFiles": 10,
  "completedFiles": ["file1.md", "file2.md"],
  "failedFiles": [],
  "inProgress": null
}
```

### 4. 分批翻译

每次只处理 **1-3 个文件**，避免超时：

1. **读取源文件** - 使用 `Read` 读取英文内容
2. **解析结构** - 识别 YAML frontmatter、Markdown 结构
3. **翻译内容** - 保持格式，仅翻译文本
4. **写入目标** - 使用 `Write` 保存中文版本
5. **更新进度** - 更新 `.translation-progress.json`

### 5. 翻译规范

#### YAML Frontmatter 处理

翻译以下字段（如果存在）：
- `title` → 翻译为中文标题
- `description` → 翻译为中文描述
- `name` → 通常保持英文，除非用户明确要求

保留以下字段不变：
- `version`, `date`, `author`, `tags`, `category`
- 技术标识符、代码、URL

#### Markdown 内容翻译

- **标题**：翻译，保持 `#` 层级
- **正文**：翻译为流畅的中文
- **代码块**：保留原样，不翻译
- **内联代码**：保留原样
- **链接**：保留 URL，可翻译链接文本
- **图片**：保留路径
- **列表**：保持格式
- **引用**：翻译内容，保留 `>` 标记

#### 特殊处理

- 技术术语首次出现时保留英文并加括号说明，例如："使用 Git 进行版本控制"
- 代码示例中的注释可以翻译
- 文件路径、命令、API 名称保持原样

### 6. 输出结构

```
source-dir/
├── README.md
├── guide.md
└── zh-CN/                    # 目标目录
    ├── README.md            # 中文版本
    └── guide.md
```

### 7. 中断与恢复

如果任务中断，下次运行时：
1. 读取 `.translation-progress.json`
2. 跳过 `completedFiles` 中已完成的文件
3. 从 `inProgress` 或第一个未完成文件继续

## 使用示例

**翻译单个文件：**
```
将 /docs/guide.md 翻译成中文，输出到 /docs/zh-CN/guide.md
```

**翻译整个目录：**
```
将 /docs 目录下所有 .md 文件翻译成中文
```

**指定格式批量翻译：**
```
将 /content 目录下所有 .md 和 .txt 文件翻译成中文
```

**继续未完成的翻译：**
```
继续翻译 /docs 目录，恢复上次的进度
```

## 质量检查

翻译完成后，进行以下检查：
- [ ] 所有文件已翻译
- [ ] YAML frontmatter 格式正确
- [ ] Markdown 链接正常工作
- [ ] 代码块未被翻译
- [ ] 文件编码为 UTF-8

## 进度跟踪

始终维护 `.translation-progress.json` 文件，包含：
- 任务开始时间
- 源目录和目标目录
- 总文件数
- 已完成文件列表
- 失败文件列表
- 当前正在处理的文件

当所有文件翻译完成后，可以保留或删除此文件（询问用户偏好）。
