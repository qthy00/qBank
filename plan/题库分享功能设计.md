# 题库分享功能设计方案

> 基于拷问结果整理的设计文档

---

## 一、需求概要

| 项目 | 决策 |
|------|------|
| 分享范围 | 整个题库 / 指定题目 / 章节 / 科目 |
| 分享形式 | 海报分享（内容冻结）+ 短链接分享（数据实时） |
| 分享对象 | 仅公开链接，移除定向分享 |
| 权限控制 | 查看无需登录，做题需注册登录 |
| 短链接 | 永久有效，无需密码 |
| 数据复用 | 复用 `AffiliateRecordVO` 表，扩展字段 |

## 三、API 接口设计

### 3.1 分享管理接口

```typescript
// app/api/share/index.ts

import type {
  ShareRecordVO,
  CreateShareReqVO,
  ShareContentVO
} from '~/types/share'
import { httpGet, httpPost, httpPut } from '~/composables/useHttp'

/**
 * 创建分享
 * @param data 分享创建参数
 */
export const createShare = (data: CreateShareReqVO) => {
  return httpPost('CreateShare', '/member/share/create', data)
}

/**
 * 获取我的分享列表
 */
export const getMyShares = async (query?: {
  page?: number
  size?: number
  shareForm?: 'poster' | 'link'
}) => {
  return await httpGet('MyShares', '/member/share/list', { query })
}

/**
 * 撤销分享
 * @param id 分享ID
 */
export const revokeShare = (id: number) => {
  return httpPut('RevokeShare', `/member/share/revoke/${id}`)
}

/**
 * 获取分享详情（通过短链接ID）
 * @param bizId 短链接ID
 */
export const getShareByBizId = async (bizId: string) => {
  return await httpGet('ShareDetail', `/app/share/${bizId}`)
}
```

## 五、核心功能实现逻辑

### 5.1 创建分享流程

```mermaid
用户选择内容 → 选择分享形式 → 后端生成bizId → 存储分享记录
                                    ↓
                              ┌─────────────┐
                              │ 海报分享    │ → 生成内容快照 → 调用海报生成服务 → 返回海报URL
                              └─────────────┘
                              ┌─────────────┐
                              │ 短链接分享  │ → 实时查询数据 → 返回短链接
                              └─────────────┘
```

### 5.2 访问分享流程

```mermaid
访问 /share/[bizId] → 查询分享记录 → 检查状态（是否撤销）
                                    ↓
                              ┌─────────────┐
                              │ 海报分享    │ → 读取snapshotData渲染冻结内容
                              └─────────────┘
                              ┌─────────────┐
                              │ 短链接分享  │ → 实时查询原表渲染最新内容
                              └─────────────┘
                                    ↓
                              展示题目内容 → 用户点击"做题"
                                    ↓
                              未登录? → 跳转登录页 → 返回继续
                              已登录? → 直接进入做题模式
```

### 5.3 bizId 生成策略

```typescript
/**
 * 生成短链接ID（8位随机字符串，大小写+数字）
 */
function generateBizId(): string {
  const chars = 'ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789'
  let result = ''
  for (let i = 0; i < 8; i++) {
    result += chars.charAt(Math.floor(Math.random() * chars.length))
  }
  return result
}
```

## 九、注意事项

1. **海报生成**：建议在服务端生成海报图片，避免客户端性能问题
2. **短链接防刷**：虽然是公开链接，但仍需考虑基础限流防止恶意刷取
3. **内容快照**：海报分享的内容快照建议使用 JSON 格式存储，便于扩展
4. **SEO**：分享页面建议支持服务端渲染，便于搜索引擎收录
